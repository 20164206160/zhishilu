## 简介
本文件上传系统基于 Spring Boot 构建，提供单文件与批量文件上传能力，以及专门的头像上传功能。系统通过本地文件系统进行物理存储，同时结合 Elasticsearch 实现元数据与检索能力。系统采用统一返回结构与全局异常处理，确保接口一致性与可维护性；通过静态资源映射与多种下载接口，实现文件访问与分发。

## 项目结构
- 控制层：负责接收 HTTP 请求并调用服务层，返回统一结构的响应。
- 服务层：封装上传、删除、路径解析等核心逻辑，包含文件类型、大小、命名与路径管理。
- 配置层：跨域、静态资源映射、拦截器注册等 Web 层配置。
- 异常处理：集中处理业务异常、认证与授权异常、参数校验异常等。
- 配置文件：应用端口、上传参数、ES 连接、JWT、日志级别等。

## 架构总览
系统采用"本地文件存储 + Elasticsearch 元数据存储"的分离设计：
- 本地存储：文件以日期分目录方式保存，使用 UUID 生成唯一文件名，避免冲突与覆盖。
- Elasticsearch 存储：仅存储文件路径（相对路径）及业务实体元数据，不直接存储文件内容，降低 ES 负载并提升扩展性。
- 头像专用存储：头像文件单独存储在独立目录中，使用用户ID+时间戳命名，支持快速访问。

### 文件上传控制器（FileController）
- 单文件上传：接收 multipart/form-data，调用服务层上传，返回相对路径。
- 批量上传：接收 files 数组，循环调用上传，返回路径列表。
- 图片访问：通过 `/file/img/**` 路径直接访问图片，无需额外处理。
- 头像访问：通过 `/file/avatar/{fileName}` 访问头像文件。
- 下载接口：根据相对路径拼装绝对路径，读取文件并设置内容类型，不存在时返回 404。

### 文件服务（FileService）
- 初始化：从配置加载上传根目录、允许类型、最大大小；确保目录存在。
- 基础上传流程：
  - 校验文件非空与大小上限（10MB）。
  - 解析扩展名并匹配允许类型列表（jpg,jpeg,png,gif,webp,bmp）。
  - 使用 UUID 生成唯一文件名，按"yyyyMMdd"组织日期子目录。
  - 创建目录并保存文件，返回相对路径。
- 头像上传流程：
  - 校验文件非空与大小上限（2MB）。
  - 解析扩展名并匹配允许类型列表（jpg,jpeg,png,gif,webp）。
  - 使用 userId_时间戳生成唯一文件名。
  - 保存到头像目录，返回文件名。
- 删除功能：支持删除普通文件和头像文件。
- 路径解析：将相对路径拼接到上传根目录，返回绝对路径。

## 性能考量
- 本地存储与 ES 分离：上传文件直接写入磁盘，ES 仅存路径与元数据，降低 ES 写入压力与网络开销。
- 日期分目录：按天组织目录，减少单目录文件数量，提升文件系统性能与可维护性。
- UUID 命名：避免同名冲突，支持并发上传场景。
- 头像专用存储：头像文件单独存储，避免与普通文件混合，提升访问效率。
- 静态资源映射：通过 WebMvcConfig 将 /file/img/** 直接映射到本地目录，减少应用层 IO 开销。
- 异步日志：操作日志拦截器在 afterCompletion 阶段异步保存，避免阻塞主请求链路。

## 故障排查指南
- 上传失败（业务异常）：检查文件是否为空、大小是否超过限制、扩展名是否在允许列表内。
- 头像上传失败：检查文件大小是否超过2MB、扩展名是否为jpg,jpeg,png,gif,webp之一。
- 图片访问404：确认路径是否正确、文件是否存在于对应日期目录下。
- 头像访问404：确认头像文件名是否正确、文件是否存在于头像目录下。
- 跨域问题：确认 CORS 配置是否允许相应来源与方法。
- 参数校验异常：查看字段校验错误信息，修正请求参数。
- 认证/授权异常：确认 JWT 头部与权限是否正确。

### API 接口规范
- 上传单个文件
  - 方法：POST
  - 路径：/file/upload
  - 参数：multipart/form-data，字段名为 file
  - 成功响应：data 为 FileUploadResp 对象，包含 path 字段
- 批量上传文件
  - 方法：POST
  - 路径：/file/upload/batch
  - 参数：multipart/form-data，字段名为 files（数组）
  - 成功响应：data 为 FileUploadResp 对象，包含 paths 字段（字符串数组）
- 图片访问
  - 方法：GET
  - 路径：/file/img/**（直接访问）
  - 参数：无
  - 成功响应：文件流，Content-Type 根据文件类型推断
  - 失败响应：404 Not Found（文件不存在）
- 头像上传
  - 方法：POST
  - 路径：/user/avatar
  - 参数：multipart/form-data，字段名为 file
  - 成功响应：data 为 Map，包含 avatar 字段（文件名）
- 头像访问
  - 方法：GET
  - 路径：/file/avatar/{fileName}
  - 路径参数：fileName（头像文件名）
  - 成功响应：头像文件流
  - 失败响应：404 Not Found（文件不存在）
- 下载文件（兼容接口）
  - 方法：GET
  - 路径：/file/download/**
  - 查询参数：path（相对路径）
  - 成功响应：文件流，Content-Type 根据文件类型推断
  - 失败响应：404 Not Found（文件不存在）

### 安全与合规建议
- 文件类型白名单：当前支持常见图片类型，建议在生产环境扩展或引入更严格的 MIME 类型检测。
- 大小限制：默认 10MB（普通文件），2MB（头像文件），建议根据业务场景调整并配合前端预检。
- 路径安全：下载接口使用相对路径拼接绝对路径，避免路径穿越；建议在生产环境限制可访问路径范围。
- 头像管理：头像上传时自动删除旧头像，防止磁盘空间无限增长。
- 静态资源访问：通过 /file/img/** 映射到本地目录，注意仅暴露必要路径，避免泄露内部结构。
- 合规与审计：结合操作日志拦截器记录访问行为，定期审计异常与失败请求。

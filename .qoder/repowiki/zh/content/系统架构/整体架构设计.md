## 引言
本项目为"知拾录"个人知识收藏管理系统，现已升级为完整的前后端分离架构。后端采用Spring Boot 3.2.1 + Java 17构建，结合Apache Shiro与JWT实现认证鉴权，使用Elasticsearch进行内容检索与存储，提供文章管理、用户认证、文件上传与下载、操作日志等能力。前端采用Vue 3 + Vite + Pinia + Vue Router构建现代化单页应用(SPA)，通过Axios进行API通信。系统遵循分层架构模式（Controller-Service-Repository），通过统一返回封装、拦截器与过滤器实现横切关注点，具备良好的可扩展性与可维护性。

## 项目结构
系统采用前后端分离的现代化项目结构，主要分为以下层次：

**后端服务层：**
- 表现层（Controller）：负责HTTP请求接入与响应封装
- 业务层（Service）：承载核心业务逻辑与领域模型编排
- 数据访问层（Repository）：基于Spring Data Elasticsearch的仓储接口
- 领域模型（Entity）：Elasticsearch文档映射实体
- 配置与安全（Config/Shiro）：Shiro过滤链与JWT集成
- 工具与通用类（Common/Util）：统一返回、分页结果、JWT工具、用户上下文

**前端应用层：**
- 视图层（Views）：Vue组件构成的用户界面
- 路由层（Router）：Vue Router实现的前端路由管理
- 状态管理层（Stores）：Pinia实现的状态管理
- 工具层（Utils）：Axios封装的API请求和工具函数
- 组件层（Components）：可复用的UI组件

## 前后端分离架构

### 前端架构设计
前端采用Vue 3 + Vite + Pinia + Vue Router + Axios的现代化技术栈：

**技术栈选择：**
- Vue 3：最新版本的渐进式JavaScript框架，提供更好的性能和开发体验
- Vite：快速的构建工具，提供即时的热重载和更快的冷启动
- Pinia：Vue官方推荐的状态管理库，比Vuex更轻量级
- Vue Router：官方路由管理器，支持嵌套路由和懒加载
- Axios：功能强大的HTTP客户端，支持拦截器和请求/响应转换

**架构特点：**
- 单页应用(SPA)：通过Vue Router实现前端路由，避免页面刷新
- 组件化设计：可复用的UI组件和业务组件分离
- 状态管理：使用Pinia管理全局状态，支持持久化存储
- API通信：通过Axios封装统一的API请求和响应处理

## 性能考虑
- **查询优化：**
  - 使用Elasticsearch的IK分词器对标题与正文进行中文分词，提升检索准确性
  - 列表查询使用NativeQuery与排序，避免全表扫描
  - 聚合查询统计常用类别，减少复杂计算开销
  - **新增**：Redis缓存热门数据和用户会话信息
- **存储策略：**
  - 图片等二进制资源存储于本地文件系统，Elasticsearch仅存路径，降低索引体积
  - 上传目录按日期分层，便于清理与管理
  - **新增**：Redis缓存热点数据，减少数据库压力
- **并发与线程：**
  - 使用异步配置（AsyncConfig）处理耗时任务（如日志写入），避免阻塞主线程
  - **新增**：Redis连接池配置，提高缓存访问性能
- **缓存策略：**
  - **新增**：Redis作为多级缓存，包括用户会话缓存和热点数据缓存
  - 缓存失效策略和一致性保证
- **前端性能：**
  - **新增**：Vue 3的Composition API提供更好的性能
  - **新增**：Vite的快速热重载和按需编译
  - **新增**：组件懒加载和代码分割
- **可扩展性：**
  - **新增**：微服务架构支持水平扩展
  - **新增**：容器化部署支持弹性伸缩

## 故障排除指南
- **认证失败：**
  - 检查请求头Authorization格式是否符合Bearer前缀
  - 确认JWT密钥与过期时间配置正确
  - 查看Shiro过滤器是否正确拦截与放行
  - **新增**：检查前端Axios拦截器是否正确添加Token

  - **新增**：检查Redis连接配置和连接池设置
- **前端路由问题：**
  - **新增**：检查Vue Router配置和路由守卫逻辑

  - **新增**：检查CORS配置和代理设置

## 结论
"知拾录"系统通过前后端分离的现代化架构设计，实现了从简单原型到完整个人知识管理系统的重大升级。系统采用Vue 3 + Spring Boot的全栈技术栈，结合微服务设计理念，提供了完整的认证授权、内容管理、文件处理与全文检索能力。

**架构优势：**
- **前后端分离**：前端SPA提供优秀的用户体验，后端API提供稳定的业务逻辑
- **微服务架构**：模块化设计支持独立部署和扩展
- **现代化技术栈**：采用最新版本的技术确保系统先进性和可维护性
- **性能优化**：Elasticsearch + Redis的组合提供高性能的数据存储和缓存
- **安全可靠**：JWT + Shiro的安全体系保障系统安全

**未来发展：**
系统具备良好的可扩展性，未来可以在以下方面进一步增强：
- **监控告警**：集成Prometheus + Grafana实现全面的系统监控
- **分布式追踪**：使用OpenTelemetry实现全链路性能追踪
- **容器化部署**：Docker + Kubernetes实现自动化部署和扩缩容
- **API网关**：引入API Gateway统一管理服务路由和安全策略
- **消息队列**：引入RabbitMQ/Kafka处理异步任务和事件驱动架构

通过持续的技术演进和架构优化，"知拾录"系统将能够支撑更大规模的业务场景，为用户提供更加优质的知识管理服务。

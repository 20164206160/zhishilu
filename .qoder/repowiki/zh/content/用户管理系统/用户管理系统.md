## 项目概述

知拾录是一个基于Spring Boot构建的个人知识收藏管理系统，采用现代化的技术栈实现用户认证与权限控制。系统采用前后端分离架构，通过JWT令牌实现无状态认证，结合Apache Shiro进行权限管理。

### 核心技术栈
- **后端框架**: Spring Boot 3.2.1 + Spring MVC
- **认证授权**: Apache Shiro + JWT + Redis Token管理
- **数据存储**: Elasticsearch 8.x + Redis
- **开发工具**: Lombok
- **语言版本**: Java 17

### 主要功能特性
- 用户注册/登录认证
- 基于JWT的无状态会话管理
- 基于Shiro的权限控制
- **管理员功能**：用户授权管理、用户列表管理、**密码重置管理**
- Elasticsearch全文搜索
- 文件上传管理
- 操作日志记录

### 字段详细说明

| 字段名 | 类型 | 索引类型 | 描述 | 约束条件 |
|--------|------|----------|------|----------|
| id | String | ID | 用户唯一标识符 | UUID生成，主键 |
| username | String | Keyword | 用户名 | 唯一约束，3-20字符 |
| password | String | Keyword | 加密后的密码 | 不可索引，长度32字符 |
| nickname | String | Keyword | 昵称 | 可选，最大20字符 |
| email | String | Keyword | 邮箱地址 | 可选，邮箱格式验证 |
| avatar | String | Keyword | 头像路径 | 可选，文件存储路径 |
| status | Integer | Integer | 用户状态 | 1-正常，0-禁用，默认1 |
| authorized | Integer | Integer | 授权状态 | 1-已授权，0-未授权，默认0 |
| createdTime | LocalDateTime | Date | 创建时间 | 自动设置 |
| lastLoginTime | LocalDateTime | Date | 最后登录时间 | 自动更新 |

### Elasticsearch映射配置

用户实体在Elasticsearch中的映射配置确保了：
- **索引优化**: 关键字字段使用Keyword类型便于精确匹配
- **搜索性能**: 时间字段使用Date类型支持范围查询
- **安全性**: 密码字段不建立索引，防止明文泄露
- **授权查询**: authorized字段支持按授权状态快速查询

## API接口规范

系统提供RESTful API接口，遵循统一的响应格式和错误处理机制。**更新** 新增管理员用户管理相关接口，包括密码重置功能。

### 认证接口

#### 用户注册

**请求URL**: `POST /api/auth/register`

**请求头**:
```
Content-Type: application/json
Accept: application/json
```

**请求参数**:
| 参数名 | 类型 | 必填 | 描述 | 长度限制 |
|--------|------|------|------|----------|
| username | String | 是 | 用户名 | 3-20字符 |
| password | String | 是 | 密码 | 6-32字符 |
| nickname | String | 否 | 昵称 | 最大20字符 |
| email | String | 否 | 邮箱地址 | 邮箱格式 |

**请求示例**:
```json
{
  "username": "john_doe",
  "password": "password123",
  "nickname": "John Doe",
  "email": "john@example.com"
}
```

**响应格式**:
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": "uuid-string",
    "username": "john_doe",
    "nickname": "John Doe",
    "email": "john@example.com",
    "avatar": null,
    "status": 1,
    "authorized": 0,
    "createdTime": "2024-01-01T12:00:00",
    "lastLoginTime": null
  },
  "timestamp": 1704067200000
}
```

### 密码安全策略

系统采用以下密码安全措施：

1. **哈希算法**: SHA-256
2. **迭代次数**: 1024次
3. **固定盐值**: zhishilu
4. **存储方式**: 明文存储哈希值，不存储原始密码
5. **密码重置安全**: 管理员重置密码后强制用户重新登录

### 管理员权限控制

管理员具有特殊权限，包括：
1. **用户管理**: 查看所有用户、管理用户授权状态
2. **用户删除**: 删除普通用户（不能删除管理员）
3. **强制登出**: 取消授权或删除用户时强制登出
4. **密码重置**: 重置普通用户的密码并强制其重新登录
5. **权限验证**: 所有管理员操作都需要验证当前用户是否为管理员
6. **安全边界**: 不能重置自己的密码，不能重置其他管理员的密码

## 故障排除指南

### 常见问题诊断

#### 1. 用户名或密码错误

**症状**: 登录时返回"用户名或密码错误"

**可能原因**:
- 用户名不存在
- 密码不正确
- 用户被禁用

**解决步骤**:
1. 验证用户名是否存在于数据库
2. 检查密码哈希是否匹配
3. 确认用户状态为正常

#### 2. 用户未授权

**症状**: 登录时返回"用户未授权，请联系网站人员"

**可能原因**:
- 用户注册后未被管理员授权
- 用户授权状态为0

**解决步骤**:
1. 确认管理员用户名配置正确
2. 检查用户授权状态
3. 通过管理员接口授权用户

#### 3. JWT令牌过期

**症状**: 访问受保护资源时返回401未授权

**可能原因**:
- 令牌已过期（默认7天）
- 服务器时间不同步
- 令牌被篡改

**解决步骤**:
1. 重新登录获取新令牌
2. 检查服务器时间配置
3. 验证令牌签名完整性

#### 4. 管理员权限不足

**症状**: 访问管理员接口时返回403禁止访问

**可能原因**:
- 当前用户不是管理员
- 管理员配置不正确
- 尝试执行不允许的操作（如重置管理员密码）

**解决步骤**:
1. 检查application.yml中的admin.usernames配置
2. 确认当前登录用户在管理员列表中
3. 重新登录确认权限
4. 遵守密码重置规则：不能重置管理员密码，不能重置自己的密码

#### 5. Elasticsearch连接失败

**症状**: 注册或登录时报数据库连接错误

**可能原因**:
- Elasticsearch服务未启动
- 网络连接问题
- 认证凭据错误

**解决步骤**:
1. 检查Elasticsearch服务状态
2. 验证连接配置
3. 确认网络连通性

#### 6. 密码重置失败

**症状**: 调用密码重置接口时返回错误

**可能原因**:
- 当前用户不是管理员
- 目标用户是管理员（不能重置管理员密码）
- 新密码格式不符合要求（长度小于6位）
- 目标用户不存在

**解决步骤**:
1. 确认当前用户具有管理员权限
2. 确认目标用户不是管理员
3. 验证新密码长度至少6位
4. 确认目标用户ID有效

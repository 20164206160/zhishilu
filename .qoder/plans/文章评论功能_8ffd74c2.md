# 文章评论功能实现计划

## 一、后端实现

### 1. 新增评论实体 `Comment.java`
路径：`src/main/java/com/zhishilu/entity/Comment.java`
- 字段：`id`、`articleId`、`content`、`createdBy`、`creatorId`、`creatorAvatar`、`parentId`（回复哪条根评论）、`replyToId`（回复的具体评论）、`replyToUser`（被回复人用户名）、`createdTime`、`likeCount`
- ES 索引：`zhishilu_comment`，`@Document(indexName = "zhishilu_comment")`

### 2. 新增 ES 索引映射 `comment-mapping.json`
路径：`src/main/resources/comment-mapping.json`
- articleId/creatorId/parentId/replyToId 为 keyword
- content 为 text（ik_max_word）
- createdTime 为 date

### 3. 新增评论仓储 `CommentRepository.java`
路径：`src/main/java/com/zhishilu/repository/CommentRepository.java`
- 继承 `ElasticsearchRepository<Comment, String>`
- 方法：`findByArticleId`、`findByParentId`、`findByArticleIdAndParentIdIsNull`

### 4. 新增请求对象 `CommentCreateReq.java`
路径：`src/main/java/com/zhishilu/req/CommentCreateReq.java`
- 字段：`articleId`（必填）、`content`（必填，最大500字）、`parentId`（选填）、`replyToId`（选填）、`replyToUser`（选填）

### 5. 新增响应对象 `CommentResp.java`
路径：`src/main/java/com/zhishilu/resp/CommentResp.java`
- 字段：同 Comment 实体 + `replies: List<CommentResp>`（回复列表）

### 6. 新增评论服务 `CommentService.java`
路径：`src/main/java/com/zhishilu/service/CommentService.java`
- `createComment(req, currentUser)` - 发布评论/回复
- `getCommentsByArticleId(articleId, page, size)` - 分页获取根评论（含前3条回复）
- `getRepliesByParentId(parentId, page, size)` - 展开查询某条评论的所有回复
- `deleteComment(id, currentUser)` - 删除评论（本人或管理员）
- `likeComment(id, currentUser)` - 点赞评论

### 7. 新增评论控制器 `CommentController.java`
路径：`src/main/java/com/zhishilu/controller/CommentController.java`
- `POST /comment/create` - 发布评论（jwt）
- `GET /comment/list?articleId=&page=&size=` - 获取评论列表（anon）
- `GET /comment/replies?parentId=&page=&size=` - 获取回复列表（anon）
- `DELETE /comment/delete/{id}` - 删除评论（jwt）
- `POST /comment/like/{id}` - 点赞评论（jwt）

### 8. 修改 `ShiroConfig.java`
- 添加 `/comment/list` 和 `/comment/replies` 为 anon（公开访问）

### 9. 新增 ES 索引初始化
在 `ElasticsearchIndexInitializer.java` 中添加 comment 索引的初始化逻辑（与 article 索引同模式）

## 二、前端实现

### 10. 修改 `ArticleDetailView.vue`
将现有的 mock 评论数据替换为真实 API 调用，实现：

**评论展示：**
- 调用 `GET /comment/list` 获取评论列表，显示总数
- 每条根评论展示：头像、用户名、内容（支持 emoji 渲染）、时间、点赞数
- 根评论下默认显示最多3条回复，超过3条显示"查看全部X条回复"按钮

**回复评论：**
- 点击"回复"按钮后，底部输入框切换为回复模式，显示"回复@用户名"提示
- 按 ESC 或点击取消退出回复模式

**发布评论（含表情包）：**
- 底部输入框右侧增加表情包图标按钮（😊）
- 点击弹出表情面板（使用 emoji-mart-vue 或内置常用 emoji 列表）
- 点击 emoji 插入到输入框光标位置
- 点击"发送"调用 `POST /comment/create`
- 发送成功后清空输入框、刷新评论列表

**删除评论：**
- 当前用户是评论作者或管理员时，显示删除按钮（小垃圾桶图标）
- 删除后刷新评论列表
